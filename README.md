# NightlyBuildX

NightlyBuildX now supports exactly three entrypoints:

1. `run_tests_report` - run tests and build report immediately (legacy/full mode).
2. `run_tests` - run tests only and save raw run data (no report generation).
3. `build_report` - build missing reports from raw run data.

## `run_tests_report` (full mode)

```bash
bash NightlyBuildX/scripts/run_tests_report --build <path-to-build> --tests <path-to-reg-tests> [--unittests on|off]
```

- Identical behavior to the previous `run_tests`.
- Generates plots and HTML report directly.
- Always writes output under `NightlyBuildX/runs/<timestamp>/`.

## `run_tests` (execution-only mode)

```bash
bash NightlyBuildX/scripts/run_tests --build <path-to-build> --tests <path-to-reg-tests> [--unittests on|off] [--remote]
```

- Runs unit tests (optional) and regression executions.
- Generates `results.json` with unit/regression check outcomes, but does **not** generate per-run `index.html` or plots.
- Stores raw run data under:
  - local: `NightlyBuildX/runs/<timestamp>/`
  - remote: `NightlyBuildX/runs_remote/<timestamp>/` (with `--remote`)

## `build_report` (offline report generation)

```bash
python3 NightlyBuildX/scripts/build_report
```

- Builds reports from raw run data generated by `run_tests`.
- Automatically scans all runs and builds only new/incomplete ones (you do not need `--run`).
- Skips runs that already contain `index.html`.
- Rebuilds/refreshes `NightlyBuildX/report/index.html`.

Optional:

```bash
python3 NightlyBuildX/scripts/build_report --tests <path-to-reg-tests> --run <timestamp>
python3 NightlyBuildX/scripts/build_report --tests <path-to-reg-tests> --report-root /path/to/report
```

## Git tracking policy

- `NightlyBuildX/runs/` is local-only and git-ignored.
- `NightlyBuildX/runs_remote/` is git-tracked (including report PNG plots).
